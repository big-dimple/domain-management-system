# 域名管理系统

## 项目概述

域名管理系统是一个现代化的应用程序，专为需要集中管理多个域名资产的企业和组织设计。通过直观的界面和强大的后台功能，系统帮助用户全面掌握域名状态、到期日期和使用情况，提供基于自定义规则的续费建议，并保留完整的操作历史记录。

本系统采用前后端分离架构，使用容器化技术部署，易于在各种环境中安装和使用。

## 核心功能

### 域名资产管理
- **全面的域名信息管理**：跟踪域名类型、到期日期、持有主体、解析管理方等关键信息
- **智能分类**：按照域名类型（gTLD、ccTLD、New gTLD）自动分类
- **自定义标记**：支持标记特殊价值域名或不再需要续费的域名
- **批量操作**：支持多选进行批量删除、批量评估等操作

### 数据导入导出
- **CSV格式支持**：便捷的CSV导入导出功能，支持批量域名管理
- **数据验证**：导入时自动验证数据格式，并提供详细的错误反馈
- **导入结果汇总**：清晰展示导入成功、更新和失败的数量及详情
- **标准化格式**：支持完整的域名信息字段导入导出，包括布尔值和日期的智能转换

#### CSV导入导出格式说明

系统支持以下CSV字段的导入导出（标题行必须精确匹配）：

**支持的字段：**
- **域名** (必填): 域名地址，如 example.com
- **域名类型** (可选): gTLD、ccTLD、New gTLD、其他，默认为 gTLD
- **到期日期** (必填): 格式为 YYYY-MM-DD 或其他可被解析的日期格式
- **持有者** (可选): 域名持有主体的中文名称
- **解析管理账号** (可选): 用于管理域名解析的账号
- **解析管理方** (可选): 提供域名解析服务的机构，如 Cloudflare、DNSPod
- **业务使用情况** (可选): 域名的具体使用场景，如"公司官网"、"未使用"
- **ICP证** (可选): ICP备案状态，如"无"、"京ICP备XXXXXXXX号"
- **年续费价** (可选): 域名年续费价格，如"39元"、"11 USD"
- **标记为不续费** (可选): 布尔值，可填入"是/否"、"true/false"或"1/0"
- **具有特殊价值** (可选): 布尔值，可填入"是/否"、"true/false"或"1/0"  
- **备注信息** (可选): 其他相关备注

**导入注意事项：**
- 文件必须为标准CSV格式，推荐UTF-8编码
- 标题行字段名称必须精确匹配（区分大小写）
- 重复域名将被更新而非重复创建
- 系统会自动为每个域名评估续费建议
- 导入完成后会显示成功、更新和失败的详细统计

**导出说明：**
- 导出文件包含所有域名的完整信息
- 布尔值将转换为"是/否"格式
- 日期将格式化为YYYY-MM-DD格式
- 文件名自动包含导出时间戳

### 智能续费建议
- **自动评估**：基于预设规则自动为每个域名生成续费建议
- **多级建议**：提供"建议续费"、"可不续费"、"请示领导"、"待评估"和"不续费"五种类型的建议
- **评估理由**：每条建议附带评估理由，帮助决策
- **自定义规则**：可根据业务需求调整评估规则

#### 续费建议规则详解

系统按以下优先级自动评估域名续费建议：

1. **不续费**：域名已被明确标记为"不续费"
2. **建议续费**：域名标记为"具有特殊价值"
3. **建议续费**：域名有实际业务使用（非"未使用"、"闲置"、"无"）
4. **建议续费**：域名有ICP证（非"无"）
5. **可不续费**：gTLD/New gTLD类型且未使用
6. **请示领导**：中国ccTLD（.cn、.中国）且未使用
7. **可不续费**：其他国家ccTLD且未使用
8. **待评估**：其他情况，需人工判断

### 数据可视化
- **实时仪表盘**：直观展示域名类型分布、续费建议分布和到期趋势
- **关键指标统计**：总域名数、即将到期域名数等重要指标一目了然
- **紧急提醒**：突出显示30天内需要续费的关键域名

### 历史记录追踪
- **完整操作日志**：记录所有域名的创建、更新、删除等操作
- **变更详情**：详细记录每次操作变更的具体字段及前后值
- **可搜索筛选**：支持按域名、操作类型等条件筛选历史记录

### 系统功能
- **健康状态监控**：实时监控服务器和数据库状态
- **自动检查**：支持手动/自动检查所有域名情况并更新续费建议
- **数据备份**：内置数据备份功能，确保数据安全

## 安装指南

### 系统要求
- 操作系统: Linux、macOS 或 Windows (支持Docker)
- Docker 20.10.0+
- Docker Compose 2.0.0+
- 开放端口: 8080(前端访问)、3001(API服务，可选)
- 至少1GB可用内存和10GB磁盘空间

### 快速安装

本项目的安装过程涉及使用一系列脚本（`01_...sh` 至 `13_...sh`）来生成完整的应用程序代码和配置。主安装脚本 `13_install_all.sh` 负责协调此过程。

1.  **获取安装脚本**

    首先，请确保您已获得所有安装脚本（`01_install_dependencies.sh` 至 `13_install_all.sh`）以及此 `readme.md` 文件。
    这些脚本通常会一起提供，例如在一个 Git 仓库或一个压缩包中。

    假设您已将这些脚本放置在一个目录中，请导航到该目录：
    ```bash
    # 示例：如果脚本位于名为 domain-management-generator 的目录中
    # cd domain-management-generator
    ```

2.  **运行主安装脚本 (`13_install_all.sh`)**

    脚本 `13_install_all.sh` 将引导您完成整个安装和项目生成过程。它会调用其他模块化脚本来创建所需的目录结构、配置文件和源代码。

    首先，赋予主安装脚本执行权限：
    ```bash
    chmod +x 13_install_all.sh
    ```
    然后运行它：
    ```bash
    ./13_install_all.sh
    ```
    该脚本支持通过命令行参数进行自定义和无人值守安装。要查看所有可用选项，请运行：
    ```bash
    ./13_install_all.sh --help
    ```
    例如，要在无人值守模式下将项目安装到 `/opt/my-domain-app` 目录，并使用自定义的MongoDB凭据：
    ```bash
    ./13_install_all.sh --non-interactive --install-dir=/opt/my-domain-app --mongo-user=myuser --mongo-password=mypassword
    ```

3.  **遵循安装向导的指引**

    主安装脚本将会：
    -   提示您（或根据参数/默认值）输入项目安装目录和MongoDB凭据。
    -   安装系统依赖（如 Docker, Docker Compose），此步骤可能需要您输入 `sudo` 密码。
    -   在您指定的项目安装目录中生成完整的项目文件。
    -   创建用于管理（构建、启动、停止等）已生成应用的管理脚本（如 `run.sh`, `stop.sh`）。

4.  **访问和管理生成的系统**

    安装完成后：
    -   项目文件将位于您在步骤 3 中指定的安装目录（例如 `~/domain-management-system` 或 `/opt/my-domain-app`）。
    -   请进入该项目目录：
        ```bash
        cd <您的项目安装目录>
        ```
    -   您可以使用该目录下的 `run.sh` 脚本来一键构建并启动应用：
        ```bash
        ./run.sh
        ```
    -   启动后，您可以通过浏览器访问前端界面：`http://<服务器IP或localhost>:8080`

    生成的项目目录中还会包含其他管理脚本，如 `stop.sh`, `logs.sh`, `reset.sh` 等，方便您进行日常维护。

## 使用指南

### 基本操作

#### 域名管理
1. **添加域名**：点击"添加域名"按钮，填写域名信息
2. **编辑域名**：在域名列表中点击编辑图标
3. **删除域名**：在域名列表中点击删除图标
4. **批量操作**：选择多个域名后，可进行批量删除或评估

#### 数据导入导出
1. **导入CSV**：
   - 点击"选择CSV文件"选择符合格式的CSV文件
   - 点击"开始导入"进行数据导入
   - 查看导入结果统计
2. **导出CSV**：
   - 点击"导出为CSV"按钮
   - 系统会自动下载包含所有域名信息的CSV文件

#### 筛选和搜索
- 使用顶部搜索框进行关键词搜索
- 点击"高级筛选"进行多条件筛选
- 支持按域名类型、持有者、续费建议等维度筛选

#### 续费建议
- 系统会自动为每个域名生成续费建议
- 可手动重新评估单个或多个域名
- 在系统设置中可查看详细的评估规则

### 系统维护

#### 健康检查
- 访问"系统状态"页面查看系统运行状况
- 监控数据库连接状态和服务器性能指标

#### 数据备份
- 在"系统状态"页面点击"开始备份"
- 备份文件会保存在服务器的备份目录中

## 常见问题

### 安装相关

**Q: 安装过程中提示Docker权限错误怎么办？**
A: 确保当前用户已添加到docker组，可以运行 `sudo usermod -aG docker $USER` 然后重新登录。

**Q: 安装脚本执行失败怎么办？**
A: 检查脚本日志，确保系统满足依赖要求，可以尝试单独运行失败的脚本进行调试。

### 使用相关

**Q: CSV导入失败怎么办？**
A: 确保CSV文件格式正确，字段名精确匹配，编码为UTF-8。检查导入结果中的错误详情。

**Q: 域名续费建议不准确怎么办？**
A: 检查域名的业务使用情况、ICP状态等信息是否填写正确，可以手动重新评估。

**Q: 忘记MongoDB密码怎么办？**
A: 查看项目目录下的 `.env` 文件，或重新运行初始化脚本重设密码。
